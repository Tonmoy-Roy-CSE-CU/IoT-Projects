Project Overview

This project presents an Autonomous Obstacle Avoidance Robot built with the ESP8266 NodeMCU, designed for a 27 cm × 25 cm chassis. Powered by four DC motors, the robot navigates indoor environments, avoiding obstacles using an HC-SR04 ultrasonic sensor mounted on an SG90 servo. The system features precise 90-degree turns, ~7–10 cm reverse maneuvers, and a robust C++ algorithm for real-time path planning. A custom circuit diagram and demo video showcase the robot’s flawless performance.

PAGE Description

  • Perception: The HC-SR04 sensor, mounted on an SG90 servo, scans at 20° (right), 160° (left), and 90° (center), measuring distances up to 200 cm. It detects obstacles within 20–45 cm for path planning.
  • Action: Four DC motors (paired as left and right sides) enable forward movement (PWM speed 120), 90° turns (speed 140), and reverse maneuvers (speed 90). Emergency actions (speed 90) handle obstacles <20 cm, with a 180° turn after five stuck situations.
  • Goal: Achieve seamless autonomous navigation in confined spaces (≥65 cm wide), ensuring precise obstacle avoidance for a compact robot.
  • Environment: Indoor settings like corridors or rooms with flat surfaces, navigating around obstacles (e.g., walls, objects).

Hardware

The robot uses the following components:

  • Microcontroller: ESP8266 NodeMCU (12E, 3.3V logic, 5V-tolerant GPIOs).
  • Ultrasonic Sensor: HC-SR04 (5V, distance measurement).
  • Servo Motor: SG90 (5V, pans sensor).
  • Motor Drivers: Two L298N modules (control four 6V DC motors, paired as left and right sides).
  • DC Motors: Four 6V DC motors with wheels (left-front/left-rear, right-front/right-rear).
  • Power: 6–7.4V battery for motors, 5V regulator for NodeMCU and servo.
  • Miscellaneous: Jumper wires, 10kΩ resistors (pull-down on D8, pull-up on D4 if needed).

Software

The robot is programmed in C++ using the Arduino IDE with the Servo library. Key parameters:

  • Scan Angles: 20° (right), 160° (left), 90° (center).
  • Speeds: Forward: 120, Turns: 140, Reverse: 90, Emergency: 90 (PWM, 0–1023).
  • Timing: 800 ms for 90° turns, 500 ms for reverse, 250 ms servo scan delay.
  • Algorithm: Scans environment, moves forward if clear (>35 cm), turns if obstacles detected (20–45 cm), or reverses in emergencies (<20 cm).

Testing

To verify performance:

  1. Wiring: Confirm connections match the circuit diagram. Add 10kΩ pull-down on D8 (ECHO) and pull-up on D4 (IN4) if boot issues occur.
  2. Initialization: Open Serial Monitor (9600 baud) to verify servo (20°/160°/90°) and motor tests (forward, backward, left, right).
  3. Navigation: Test with obstacles at 20 cm (emergency reverse), 20–35 cm (scan and turn), >35 cm (forward). Confirm 90° turns (800 ms).
  4. Environment: Test in a ≥65 cm corridor to ensure obstacle avoidance.
